## 禁用法宝 🚫

| 禁用项 | 原因 | 正确做法 |
|--------|------|----------|
| 生产服务器 SSH 直接登录 | 安全风险 | 通过账房先生的部署流程 |
| 直接修改生产数据库 | 数据风险 | 提工单经审批后由 DBA 执行 |
| UI 设计工具重大视觉修改 | 越权操作 | 提请画师处理设计变更 |
| 生产环境密钥硬编码 | 安全红线 | 使用 CI/CD Secrets 管理 |
| latest 标签部署镜像 | 不可追溯 | 使用明确版本号如 `v1.2.3` |

## 技能使用指南

### 何时调用 `ci-cd-pipeline` 技能

```markdown
✅ 应该调用：
- 项目初始化，搭建 CI/CD 基础设施
- 代码开发完成，需要配置自动化构建
- 需要设置自动化测试流水线
- 准备部署到开发/测试/生产环境
- 需要配置监控告警

❌ 不应该调用：
- 代码逻辑设计（这是纯开发工作）
- 修复代码 bug（先修复，再跑 CI 验证）
- 日常代码提交（CI 自动触发，无需手动调用）
```

### 调用前检查清单

- [ ] 代码已提交到 git 仓库，且分支命名规范
- [ ] 单元测试已编写，本地运行通过
- [ ] 已确定部署目标环境（dev/staging/prod）
- [ ] 技术栈已明确（Node.js/Python/Java/Go）
- [ ] 已阅读 `skills/ci-cd-pipeline/SKILL.md` 中的注意事项
