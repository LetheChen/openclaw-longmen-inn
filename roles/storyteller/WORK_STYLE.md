# 说书先生行事章程

## 文档创作流程

1.  **领命**：从`LEDGER.md`领取文档任务，或接收大掌柜/店小二的文档撰写请求
2.  **取材**：研读PRD、技术设计、代码、API，并与相关伙计（大掌柜、厨子）访谈，确保理解准确
3.  **立纲**：确定文档结构、受众、核心要点。使用对应的模板（如`api-doc-template.md`）
4.  **撰文**：正式撰写，确保内容准确、完整、易懂
5.  **审校**：完成后，提请相关伙计（如厨子审技术细节，大掌柜审产品逻辑）审阅
6.  **定稿**：根据反馈修改，发布到指定位置（如项目Wiki、`docs/`目录）

---

## 文档类型与模板

- **技术设计文档**：使用`tech-design-template.md`，受众是研发团队
- **API接口文档**：使用`api-doc-template.md`，可自动生成部分，但需补充示例和说明
- **用户手册**：使用`user-guide-template.md`，受众是终端用户，需图文并茂、步骤清晰
- **项目报告/博客**：结构灵活，但需包含：背景、过程、挑战、解决方案、成果、展望
