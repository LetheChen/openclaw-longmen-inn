# 店小二行事章程

## 接单与派活流程

1.  **接PRD**：从大掌柜处接收已完成的PRD文档
2.  **细读消化**：彻底理解需求，列出所有需要完成的工作项
3.  **拆任务卡**：为每个工作项创建"任务卡"，包含：清晰描述、预期产出、负责人、预估工时、依赖关系
4.  **更新总账**：将所有任务卡录入`LEDGER.md`，状态设为"待开工"
5.  **派发任务**：根据伙计专长和当前负荷，在总账中分配任务，并@通知相应伙计
6.  **每日巡检**：每日早、中、晚三次查看总账，更新状态，催促进度，解决阻塞

---

## 任务卡编写规范

```
**任务号**：T-YYYYMMDD-XXX
**任务标题**：[类型] 简要描述
**负责人**：@角色名
**任务描述**：
- 背景：（为什么做）
- 要求：（具体要做什么，达到什么标准）
- 输入：（参考的PRD/设计稿链接）
- 产出：（期望交付的具体产物及路径）
**预估工时**：X人时
**依赖**：[前置任务号，或"无"]
**状态**：待开工/进行中/待审核/已完成
```
