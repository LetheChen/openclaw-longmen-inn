# 龙门客栈业务管理系统 - 产品需求文档 (PRD)

> **版本**: v1.0  
> **日期**: 2026-03-14  
> **编写**: 大掌柜 (诸葛掌柜)  
> **审核**: 老板娘 (凤老板)

---

## 1. 项目背景与目标

### 1.1 背景
基于 OpenClaw 多 Agent 协作平台，构建"龙门客栈"业务管理系统，实现7个角色（大掌柜、店小二、厨子、账房先生、说书先生、画师、老板娘）的协作管理。

### 1.2 目标
- 建立统一的多 Agent 协作管理平台
- 实现任务全生命周期管理（创建→分配→执行→审核→完成）
- 建立龙门令功勋体系，激励 Agent 积极性
- 提供实时监控看板，可视化展示项目进度

## 2. 用户角色与权限

### 2.1 系统用户角色

| 角色 | 说明 | 权限 |
|------|------|------|
| **系统管理员** | 平台运维人员 | 所有权限 |
| **项目经理** | 项目负责人 | 项目/任务/Agent管理 |
| **普通用户** | 项目成员 | 查看任务、更新进度 |

### 2.2 Agent 角色（系统内置）

| Agent | 职责 | 权限范围 |
|-------|------|----------|
| 大掌柜 | 战略决策、PRD输出 | 项目创建、任务分配 |
| 店小二 | 任务拆解、进度跟踪 | 任务管理、状态更新 |
| 厨子 | 编码实现 | 任务执行、代码提交 |
| 账房先生 | 代码审查、记功 | 审核任务、发放龙门令 |
| 说书先生 | 文档编写 | 文档任务、报告输出 |
| 画师 | UI/UX设计 | 设计任务、原型输出 |
| 老板娘 | 总控协调 | 全局监控、仲裁协调 |

## 3. 功能需求

### 3.1 实时看板 (Dashboard)

#### 3.1.1 任务统计卡片
- 显示今日任务总数、进行中、已完成、阻塞数量
- 本周/本月任务趋势图表
- 各 Agent 任务负载分布

#### 3.1.2 Kanban 看板
- 按状态分栏展示：待开工 → 进行中 → 待审核 → 已完成
- 拖拽式任务状态变更
- 任务卡片显示：编号、标题、负责人、优先级、截止时间
- 快速筛选：按项目、按 Agent、按优先级

#### 3.1.3 实时活动流
- 显示最近系统活动：任务创建、状态变更、龙门令发放
- 支持按 Agent 过滤

### 3.2 任务管理

#### 3.2.1 任务创建
- 表单字段：
  - 任务标题（必填）
  - 任务描述（支持 Markdown）
  - 所属项目（下拉选择）
  - 执行 Agent（下拉选择）
  - 优先级（紧急/高/中/低）
  - 预计工时（小时）
  - 截止时间
  - 附件上传
- 自动生成任务编号（格式：T-YYYYMMDD-NNN）

#### 3.2.2 任务详情
- 完整任务信息展示
- 任务状态流转历史（时间线形式）
- 关联的子任务列表
- 产出物链接（代码/文档/设计稿）
- 评论区（支持 @Agent）

#### 3.2.3 任务流转
- **状态机**：待开工 → 进行中 → 待审核 → 已完成
  - 其他状态：已阻塞、已取消
- **流转规则**：
  - 执行 Agent 可将"待开工"→"进行中"、"进行中"→"待审核"
  - 审核 Agent 可将"待审核"→"已完成"或驳回至"进行中"
  - 只有店小二/老板娘可将任务标记为"已阻塞"
  - 任务完成时自动计算实际工时

### 3.3 Agent 管理

#### 3.3.1 Agent 列表
- 卡片式展示所有 Agent
- 显示信息：头像、名称、称号、当前状态（空闲/忙碌/离线）、龙门令总数、等级
- 快速操作：查看详情、编辑配置、查看任务

#### 3.3.2 Agent 详情
- **基本信息**：名称、称号、信条、职责描述
- **状态监控**：
  - 当前任务列表
  - 本周/本月任务统计
  - 平均任务完成时间
  - Token 消耗统计
- **工作历史**：最近完成的任务列表
- **配置管理**：
  - LLM 模型选择（GPT-4/GPT-3.5/Claude等）
  - 系统提示词编辑
  - 技能管理（添加/移除技能）

#### 3.3.3 Agent 性能分析
- 任务完成率趋势图
- 平均响应时间统计
- 龙门令获取趋势
- 等级晋升历史

### 3.4 龙门令排行榜

#### 3.4.1 功勋体系规则

**龙门令获取方式**：

| Agent 角色 | 龙门令获取条件 | 龙门令数量 |
|------------|----------------|------------|
| 大掌柜 | 情报搜集准确、战略决策正确、PRD质量高 | 10-50/项 |
| 店小二 | 任务拆解高效、进度跟踪及时、协调成功 | 5-30/项 |
| 厨子 | 代码质量高、一次通过审查、帮助他人 | 10-40/任务 |
| 账房先生 | 审查公正准确、龙门令发放及时、账本清晰 | 5-25/项 |
| 说书先生 | 文档质量高、文案打动人心、交付及时 | 5-30/篇 |
| 画师 | 设计稿质量高、规范清晰、交付及时 | 5-30/个 |
| 老板娘 | 总控协调成功、解决冲突、兜底成功 | 10-50/次 |

**等级体系**：

| 等级 | 龙门令范围 | 称号 | 特权 |
|------|------------|------|------|
| 1级 | 0-99 | 新手伙计 | 基础任务 |
| 2级 | 100-499 | 熟练工 | 可接中级任务 |
| 3级 | 500-999 | 老师傅 | 可接高级任务 |
| 4级 | 1000-2499 | 大管家 | 可分配任务给他人 |
| 5级 | 2500-4999 | 传奇掌柜 | 可参与战略决策 |
| 6级 | 5000+ | 龙门传说 | 拥有所有特权 |

#### 3.4.2 排行榜功能

**总榜**：
- 按龙门令总数排序
- 显示：排名、Agent头像、名称、等级、龙门令总数
- 支持时间范围筛选：本周、本月、本年、全部

**分角色榜**：
- 按 Agent 角色分类排行
- 展示各角色的龙门令获取情况

**龙门令明细**：
- 每个 Agent 的龙门令获取历史
- 显示：时间、任务/原因、龙门令数量
- 支持导出

**晋升记录**：
- 展示 Agent 等级晋升历史
- 晋升时间线可视化

### 3.5 OpenClaw 服务集成

#### 3.5.1 Gateway 状态监控

- **连接状态**：实时显示与 OpenClaw Gateway 的连接状态
- **服务健康**：监控 Gateway 服务健康度
- **配置信息**：显示 Gateway URL、API Key 等配置
- **操作按钮**：连接/断开、重启 Gateway

#### 3.5.2 Agent 会话管理

- **会话列表**：显示所有 Agent 的当前会话
- **会话详情**：查看会话历史、消息记录
- **会话控制**：创建新会话、结束会话、清空历史
- **实时消息**：WebSocket 实时接收 Agent 消息

#### 3.5.3 消息路由配置

- **渠道配置**：配置飞书/钉钉/邮件/短信等消息渠道
- **路由规则**：设置消息路由规则（如：任务完成通知项目经理）
- **模板管理**：配置消息模板（支持变量）
- **发送记录**：查看消息发送历史

#### 3.5.4 日志与审计

- **操作日志**：记录所有系统操作
- **API 日志**：记录 API 调用情况
- **错误日志**：记录系统错误
- **审计追踪**：关键操作的审计记录

## 4. 非功能需求

### 4.1 性能需求

- **响应时间**：页面加载 < 2s，API 响应 < 500ms
- **并发支持**：支持 100+ 并发用户
- **数据量**：支持 10000+ 任务，100000+ 日志记录

### 4.2 安全需求

- **认证授权**：JWT Token + RBAC 权限控制
- **数据加密**：敏感数据加密存储
- **输入验证**：所有输入参数校验
- **审计日志**：关键操作记录审计日志

### 4.3 可用性需求

- **系统可用性**：99.5% 以上
- **故障恢复**：支持自动重启和故障转移
- **数据备份**：定时自动备份

### 4.4 可扩展性需求

- **模块化设计**：支持功能模块热插拔
- **插件机制**：支持自定义插件开发
- **API 开放**：提供完整 RESTful API

## 5. 接口设计

### 5.1 RESTful API 规范

- **基础 URL**: `/api/v1`
- **认证方式**: Bearer Token (JWT)
- **响应格式**: JSON
- **状态码**: 遵循 HTTP 标准状态码

### 5.2 主要 API 列表

| 模块 | 接口 | 方法 | 说明 |
|------|------|------|------|
| **认证** | `/auth/login` | POST | 用户登录 |
| | `/auth/logout` | POST | 用户登出 |
| | `/auth/refresh` | POST | 刷新 Token |
| **项目** | `/projects` | GET | 获取项目列表 |
| | `/projects` | POST | 创建项目 |
| | `/projects/{id}` | GET | 获取项目详情 |
| | `/projects/{id}` | PUT | 更新项目 |
| | `/projects/{id}` | DELETE | 删除项目 |
| **任务** | `/tasks` | GET | 获取任务列表 |
| | `/tasks` | POST | 创建任务 |
| | `/tasks/{id}` | GET | 获取任务详情 |
| | `/tasks/{id}/status` | PUT | 更新任务状态 |
| | `/tasks/{id}/assign` | PUT | 分配任务 |
| **Agent** | `/agents` | GET | 获取 Agent 列表 |
| | `/agents/{id}` | GET | 获取 Agent 详情 |
| | `/agents/{id}/config` | PUT | 更新 Agent 配置 |
| | `/agents/{id}/stats` | GET | 获取 Agent 统计 |
| **龙门令** | `/longmenling/ranking` | GET | 获取排行榜 |
| | `/longmenling/{agent_id}` | GET | 获取 Agent 龙门令详情 |
| | `/longmenling/logs` | GET | 获取龙门令记录 |
| **OpenClaw** | `/openclaw/status` | GET | 获取 Gateway 状态 |
| | `/openclaw/agents` | GET | 获取 Agent 会话列表 |
| | `/openclaw/sessions/{id}` | GET | 获取会话详情 |
| | `/openclaw/messages` | POST | 发送消息 |

### 5.3 WebSocket 实时接口

| 事件 | 方向 | 说明 |
|------|------|------|
| `connect` | C→S | 客户端连接 |
| `disconnect` | C→S | 客户端断开 |
| `task:created` | S→C | 新任务创建通知 |
| `task:updated` | S→C | 任务更新通知 |
| `task:completed` | S→C | 任务完成通知 |
| `agent:status` | S→C | Agent 状态变更 |
| `message:new` | S→C | 新消息通知 |
