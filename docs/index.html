规则即秩序，协作创价值

龙门客栈通过 INN_RULES.md 定义客栈总规，明确每个角色的职责边界、绝对禁令与协作流程。AI 角色在规则约束下各司其职，有序协作。

📜 客栈五条铁规

  1. 开工先看账 — 每日上线先查阅 LEDGER.md，明确挂牌任务
  2. 各司其职 — 严格遵守角色职责，不越权、不越界
  3. 记忆传承 — 每次交班更新 MEMORY_LOG.md，接续工作
  4. 凭据归档 — 产出物存入 deliverables/，记录凭据
  5. 龙门令记功 — 工作量认定以账房先生记录为准

INN_RULES 规则引擎

通过规则文件定义协作机制，防止越权越界。明确绝对禁令，保障系统有序运转，新角色也能快速融入团队。

实时追踪任务状态与成员表现，像 Trello 一样直观。任务创建、分配、状态流转全流程可视化。

每个角色都有清晰的职责边界、协作规则与沟通风格，确保团队协作高效有序。

🐾
老板娘 · 凤老板
main · 总控协调
总揽全局，查缺补漏。做调度、管应急、控全场，是整个客栈的核心枢纽。
总控协调兜底仲裁
禁令：无事不干预伙计工作，不永久替代特定角色
🦊
大掌柜 · 诸葛掌柜
innkeeper · 战略规划
洞察先机，定夺方向。情报搜集、需求分析、PRD 输出，确保项目始终在正确方向上前行。
战略决策PRD技术选型

🏃
店小二 · 快手阿贵
waiter · 任务调度
承上启下，跑腿催办。接收 PRD，拆解任务，跟踪进度，疏通阻塞，推动流程顺畅运转。
任务拆解进度跟踪协调
禁令：不擅自修改 PRD，不替伙计做技术决策

厨子 · 神厨小福贵
chef · 编码实现
灶上功夫，代码实现。接任务、写代码、解 Bug，是客栈产出代码的核心力量。

禁令：不擅自修改需求，不提交未经自测的代码
🎨
画师 · 墨染先生
painter · UI 设计
将抽象需求转化为美观的视觉界面，设计用户交互流程，制定并维护视觉规范。
UI设计原型UX规范
禁令：不直接编写前端代码，不交付未标注的设计稿
🧮
账房先生 · 铁算盘老方
accountant · 质量把控
查验稽核，记账发赏。审查代码、验收功能、记录龙门令、管理交付，确保质量无虞。
代码审查QA龙门令交付
禁令：不擅自修改代码，不篡改龙门令记录
✍️
说书先生 · 妙笔生花
storyteller · 文档输出
妙笔生花，记录传承。撰写技术文档、用户手册、项目报告与品牌文案，传播知识与价值。
文档报告文案知识库
禁令：不编写未经确认的技术细节，不扭曲产品需求
